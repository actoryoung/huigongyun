# 工作流与目标确认

## 目标工作

基于《项目核心文档.md》的首要目标，当前阶段的目标工作是：

1. 先实现从 Excel 主元器件清单抽取逐柜 BOM 与项目汇总 BOM。
2. 保证输出包含来源说明、置信度和基础校验提示。
3. 提供可导出的 Excel / JSON 结果。

## 当前确认的工作流程

### 第 1 步：输入接入
- 接收项目文件路径或文件夹。
- 识别文件类型，优先处理 Excel 主元器件清单。

### 第 2 步：结构化解析
- 抽取柜号、柜型、规格、数量、品牌等字段。
- 保留来源引用，便于后续追溯。

### 第 3 步：柜体与 BOM 生成
- 生成逐柜 BOM。
- 汇总形成项目级 BOM。

### 第 4 步：校验
- 检查缺项、重复项、数量异常、品牌冲突等基础问题。

### 第 5 步：导出
- 输出 JSON。
- 后续扩展 Excel 导出。

## 逐个完成顺序

1. Excel 解析入口
2. 柜体与 BOM 生成
3. 校验与导出
4. 测试补齐
5. 文档更新

## 后续执行原则

- 每次只推进一个最小闭环。
- 先做可运行骨架，再补业务规则。
- 每一步都补相应测试，避免只堆代码不验证。
