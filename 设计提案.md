# 低压电气成套智能报价清单生成系统 — 设计提案

**版本**: 1.0

**摘要**:
本提案面向低压电气成套项目报价场景，提出一个可交付的系统设计与实现路线，目标是从非结构化/半结构化资料（图纸、PDF、Excel、Word、图片等）自动或半自动生成逐柜 BOM、项目汇总 BOM、报价清单与校验报告，并保证输出可追溯、支持人工修正与导出。

**关键目标**:
- 自动抽取柜体信息与逐柜 BOM
- 物料归一与相似合并
- 报价汇总与风险校验（长交期、品牌冲突等）
- 人机协同与结果可追溯

**1. 问题分析**
- 资料异质：输入可能为 DWG/PDF/图片/Excel/Word，结构不定，表格与文本混合。
- 模板不统一：不同项目/人员写法差异导致字段命名不一致与缺失。
- 信息分散：关键字段（柜号、柜型、额定电流、品牌、长交期标记）分布在不同文件和不同位置。
- 经验依赖：典型配置依赖资深工程师经验，需要沉淀为规则/样本。
- 变更频繁：资料版本迭代要支持差异比对与变更回溯。

影响：需要稳健的解析能力、可配置化的规则引擎、有效的物料规范化机制和良好的人机交互流程。

**2. 总体架构**
- 资料接入层
  - 文件管理、版本控制、任务队列
- 解析层
  - 文本抽取：OCR、文本清洗
  - 表格解析：PDF/图片表格识别或直接读取 Excel
  - 布局分析：定位柜体区域（后续迭代）
- 语义归一层
  - 物料词典与正则解析
  - 向量化与近似检索（FAISS）用于相似合并
- BOM 生成与规则引擎
  - 基于柜型/额定电流/接地方式的模板规则
  - RAG（Retriever-Augmented Generation）用于参考历史样本补全
- 校验与风险评估
  - 一致性检查、数量异常检测、品牌/长交期提示
- 人机协同层
  - 可视化编辑、来源回溯、人工修正记录
- 导出与报告层
  - Excel/JSON 导出、差异报告、审计日志

部署建议：后端 Python 服务（Flask/FastAPI），向量库 FAISS，数据库 PostgreSQL（或轻量 SQLite 开发期），前端 React/Vue。

**3. 数据模型（示例）**
- Project: id, name, files[], created_at, versions[]
- FileIndex: file_id, file_name, file_type, extracted_text, tables[], pages[], uploaded_by
- Cabinet: cabinet_id, cabinet_no, cabinet_type, rated_current, sources[], confidence
- MaterialRecord: id, cabinet_id, name, spec, unit, qty, brand, source, normalized_id, confidence, remarks
- BOMSummary: normalized_id, name_variants[], total_qty, unit, preferred_brand, price_if_available
- AuditLog: record_id, user, action, timestamp, before, after, rationale

这些对象需要同时保留来源引用（file_id、page_no、table_row_id 或 OCR bbox）以实现可追溯。

**4. 关键算法与实现方案**

- OCR 与 布局解析
  - 方案：优先使用开源 `PaddleOCR`、`Tesseract`（中文场景优先 Paddle），结合 `layoutparser` 做页面元素检测。
  - 要点：保留 OCR 字符置信度与文字 bbox，支持点击跳转显示源图（人机协同）。

- 表格抽取与 Excel 解析
  - Excel：优先直接读取（`openpyxl`/`pandas`）；若 Excel 模板混乱，做列名聚类与映射。
  - PDF 表格：使用 `pdfplumber`/`camelot`/`tabula` 做行列解析，结合后处理合并单元格。

- 字段抽取（柜号、柜型、额定电流、品牌等）
  - 规则优先：使用正则、字典与启发式规则抽取高精度字段。
  - ML/LLM 补充：使用轻量 NER 模型或 LLM（提示工程）处理模糊情形，LLM 输出需带来源与置信度说明。

- 物料归一化与相似匹配
  - 分层策略：
    1. 规则/词典匹配：常见物料名、品牌词典、型号分解规则
    2. 字符串相似度：`fuzzywuzzy` 或 `RapidFuzz` 进行模糊匹配
    3. 向量检索：使用 sentence-transformers 生成 embeddings，FAISS 做近邻检索，阈值控制合并
  - 型号解析器：拆分型号中的参数（电流、框架、极数等），用于规格归并与数量合并。

- 逐柜 BOM 生成：规则引擎 + RAG
  - 规则引擎（Drools/自建规则或简单优先级规则链）生成典型物料集合。
  - 若规则覆盖不足，RAG 检索历史相似柜体（基于柜型、额定电流、回路数向量）并提取典型 BOM 补全，人工确认后沉淀。

- 校验与异常检测
  - 一致性检测：柜号跨文件匹配（同名不同写法需归一后比对）
  - 数量异常检测：历史统计规则或简单 z-score 检测极端数量
  - 品牌冲突检测：技术说明中指定品牌优先级检查与冲突报警

- 置信度计算（合成策略）
  - 字段置信度由多个信号合成：OCR confidence、规则命中权重、相似度分数、向量检索距离
  - 输出：每条 MaterialRecord 附带 confidence ∈ [0,1] 与来源标签集合

**5. 置信度与可解释性策略**
- 所有关键决定必须保留证据链：来源文件/页/行或 OCR bbox + 匹配得分 + 规则名。
- UI 层显示三类标签：`高置信（绿色）`、`中置信（黄色）`、`低置信（红色，需人工确认）`。
- 所有人工修改将写入 `AuditLog` 并作为后续模型/规则训练样本。

**6. 测试策略与评估指标**

- 数据准备
  - 构建多模板样例集（至少 10 个项目样例覆盖不同模板）
  - 人工标注的“金标准”逐柜 BOM 以供评估

- 指标
  - 字段抽取：Precision / Recall / F1（柜号、柜型、额定电流、品牌、数量）
  - 物料归一：Top-1 准确率、合并召回率
  - 逐柜 BOM 端到端：项目级 Recall/Precision
  - 风险提醒准确性（长交期、品牌冲突检测的 Precision/Recall）
  - 性能指标：单项目平均处理时延、内存使用

- 测试用例类型
  - 模板差异（列名不同）
  - OCR 弱质量（低分辨率、噪声）
  - 缺字段（数量/品牌缺失）
  - 相似物料命名差异（别名、型号轻微变动）
  - 资料版本差异（变更检测）

- 验证流程
  - 单元测试：解析器、归一器、价格计算模块
  - 集成测试：端到端从文件到导出
  - 人工抽样审核：对低置信记录人工核验并记录结果

**7. 可视化与交互设计（建议）**

- 视图概览
  - 项目概览卡片：文件列表、处理进度、风险摘要
  - 柜体列表：显示 `柜号`、`柜型`、`额定电流`、`来源`、`置信度`、可筛选与排序
  - 单柜 BOM 展示：每行显示 `物料名`、`规格`、`数量`、`品牌`、`来源`、`置信度`，展开可查看原文或 OCR 图片片段
  - 差异对比视图：并列显示旧版本与新版本，新增/删除/数量变更高亮
  - 风险仪表盘：长交期、品牌冲突、数量异常统计图

- 交互要点
  - 点击物料可定位源（文件/页/行或 OCR bbox）
  - 支持批量确认/批量修改
  - 所有修改需填写理由（短文本）以便审计与学习

**8. 实施路线与里程碑（建议）**

- MVP（2–4 周） — 验收目标：能读取 Excel 主元器件清单，生成逐柜 BOM 与项目汇总并导出。
  - 任务：
    1. 项目与文件索引模型
    2. Excel 解析与字段映射
    3. 基础物料归一（词典+fuzzy）
    4. 逐柜 BOM 生成模板（规则引擎最小实现）
    5. 导出（Excel/JSON）

- Phase 2（4–8 周） — 增强解析能力与 UI
  - PDF/OCR、表格抽取
  - 向量检索（FAISS）用于相似物料合并
  - 基础前端界面支持人工修正

- Phase 3（8–12 周） — RAG、变更分析与风险分级
  - 历史相似柜体检索与典型 BOM 推荐
  - 版本差异自动分析
  - 风险分级模型（长交期、关键件单源风险）

**9. 风险与缓解措施**
- OCR/布局错误
  - 缓解：提供原文定位与人工纠错回路，将纠错保存用于微调
- 规则复杂度增长
  - 缓解：分层规则与优先级，保留人工覆盖权限
- 数据隐私/商用 API 成本
  - 缓解：优先开源栈，必要时对关键路径选择商用服务并做成本评估
