# 项目核心文档（必须在每次工作前读取）

## 一、总体目标
- 构建一个可运行的低压电气成套项目智能报价清单生成系统原型，能够从非结构化/半结构化资料（图纸、Excel、Word、图片等）生成逐柜 BOM、项目汇总 BOM、报价清单与校验报告；结果可解释、支持人工修正并可导出。

## 二、关键输入
- 项目图纸（DWG/PDF/图片）
- 主元器件清单（Excel）
- 技术/配置说明（Word/PDF）
- 设备清单或客户需求（Excel/Word/PDF）

## 三、核心输出
- 项目资料解析索引（文件清单、类型、识别到的约束）
- 柜体清单（柜号、柜型、额定电流、回路数、来源、置信度）
- 逐柜 BOM（物料名、规格、单位、数量、品牌、来源依据、风险提示）
- 项目汇总 BOM（合并后供采购/报价）
- 报价清单（若有价格字段则计算小计/合计）
- 校验报告（缺项/重复/数量异常/品牌冲突/长交期等）

## 四、核心功能与实现要点
- 资料解析：文件分类、文本/OCR 提取、表格解析
- 柜体抽取：从清单或图纸识别柜号、柜型、参数
- BOM 生成：规则引擎 + 历史案例/相似检索（可用 RAG）
- 物料归一：词典/映射与相似匹配合并同类项
- 报价计算：支持样例价格、按柜/标段汇总
- 校验与追溯：每条记录保留来源与置信度，支持版本差异对比
- 人机协同：界面或导出文件允许人工修改并保留修改记录

### 报价清单生成策略（MVP）
- 报价计算不前置到解析层，而是基于项目汇总 BOM 做二次计算，避免把价格逻辑与识别逻辑耦合。
- 价格来源按优先级选择：人工确认价格 > 明确价格表 > 样例价格 > 缺价占位。
- 报价输出先保留结构完整性，即使缺少价格也要生成报价行，并标注缺价提示与价格来源状态。
- 最小报价口径包括：物料单价、行小计、柜体小计、标段小计、项目总价。
- 如果价格来源不可用，只输出成本项结构和待确认提示，不进行猜测性补价。

### 报价清单落地方案
1. 在物料记录或报价记录中补充 `unit_price`、`price_source`、`price_confidence` 字段。
2. 新增报价计算模块，对汇总 BOM 逐行计算 `subtotal`，再按柜体、标段和项目汇总。
3. 导出报价清单 sheet 与 JSON 结果，缺价项单独标红或给出提示字段。
4. 增加最小测试：有价格时验证总价正确，无价格时验证结构保留和缺价提示正确。
5. 后续再扩展税率、折扣、运费、利润率、商务审批等复杂能力。

## 五、关键业务规则（最低覆盖）
- 柜号一致性：图纸、清单、配置说明可建立一一对应
- 柜型影响 BOM：不同柜型对应典型物料集合
- 额定电流影响规格：影响断路器、互感器、母排等
- 接地方式、进出线方式影响辅材配置
- 品牌优先级：技术说明指定品牌优先
- 长交期提示：在清单中单独标注并提示风险
- 人工确认优先：人工修正覆盖自动识别结果

## 六、交付物与验收要点
- 能读取示例资料并输出逐柜 BOM 与汇总 BOM（Excel/JSON）
- 输出可追溯来源与简单置信度/校验报告
- 支持人工修正并导出最终结果
- 提供运行说明与示例数据

## 七、开发/工程要求（简要）
- 数据结构需包含：项目、文件索引、柜体、物料记录、规则/来源、校验结果
- 可分阶段交付：基础（表格解析+BOM）、进阶（图纸识别+相似检索）、挑战（DWG/变更分析）

## 八、优先级与短期目标（首要）

2. 保证输出包含来源说明与校验提示
3. 提供可导出的 Excel/JSON 示例
